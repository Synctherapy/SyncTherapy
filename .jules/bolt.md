## 2025-02-28 - [Performance] Prevented Node.js event loop blocking during Next.js sitemap generation
**Learning:** `app/sitemap.ts` in Next.js was utilizing synchronous file I/O operations (`fs.readFileSync`, `fs.readdirSync`, `fs.statSync`) to parse markdown files and scan directories for routes. Synchronous I/O in server components or Next.js route handlers blocks the Node.js event loop, preventing the server from handling other requests concurrently while the sitemap is being generated. This is an architecture-specific bottleneck that causes severe latency spikes under load.
**Action:** Refactored `app/sitemap.ts` to use `fs/promises` (`fs.promises.readFile`, `fs.promises.readdir`, `fs.promises.stat`) and orchestrated the parallel execution of directory scanning and file reading using `Promise.all()`. This allows the V8 engine to offload the I/O operations to the system and continue serving other requests, drastically improving sitemap generation throughput and overall server responsiveness.
